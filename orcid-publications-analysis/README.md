# ORCID Publications Analysis

**Цель:** Извлечь и агрегировать публикации ученого из ORCID API с полями:
- 'year' — год публикации
- 'title' — название работы
- 'doi' — DOI публикации
- 'eid' — Scopus EID

**Логика обработки данных:**
- Агрегировать публикации внутри каждой ORCID group, выбирая версию с максимальным годом публикации.
- Сохранять EID, если он присутствует хотя бы в одной версии публикации.
- Не выполнять глобальную дедупликацию по названию, так как title не является уникальным идентификатором.
- Не объединять записи без DOI с записями с DOI при совпадении названия из-за отсутствия объективного критерия для склейки.

**Структура репозитория:**
- 'notebooks/orcid_publications_analysis.ipynb' — Jupyter Notebook с анализом
- 'requirements.txt' — зависимости проекта

**Запуск:**
1. Установить зависимости: 'pip install -r requirements.txt'
2. Открыть ноутбук: 'notebooks/orcid_publications_analysis.ipynb'
3. Выполнить все ячейки, финальный DataFrame доступен как 'works_df'
4. Данные выгружаются в Excel 'report.xlsx'
